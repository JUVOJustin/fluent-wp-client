## Overview

`fluent-wp-client` supports static auth and request-aware auth.

:::tip[Start simple]
Resolve auth in your app layer (middleware, request context), create a `WordPressClient` with that auth, and pass the client through your handlers.
:::

```ts title="Recommended default"
export function createWordPressClientForRequest(request: Request): WordPressClient {
  const token = readUserJwtFromRequest(request);
  return new WordPressClient({
    baseUrl: process.env.WP_BASE_URL!,
    auth: token ? { token } : undefined,
  });
}
```

## Which method should I use?

| Use case | Best option |
| --- | --- |
| Public read-only content | No auth |
| Backend jobs, admin scripts | Basic auth (application passwords) |
| SPA/mobile, external token flow | JWT auth |
| Logged-in WordPress browser session | Cookie + nonce |
| Prebuilt `Authorization` value | `authHeader` or `auth.authorization` |
| Signature/HMAC/OAuth-like | `authHeaders` provider |
| Request-scoped auth selection | `createAuthResolver` |

## Request-aware auth headers

Use `authHeaders` when auth must be computed per request (signatures, HMAC, rotating keys):

```ts title="Request signing"
const wp = new WordPressClient({
  baseUrl: 'https://example.com',
  authHeaders: ({ method, url, body }) => ({
    Authorization: createSignedAuthHeader({ method, url: url.toString(), body }),
  }),
});
```

## Resolver helpers

`createAuthResolver` returns a typed function that selects an auth shape based on context. It does not authenticate — it only picks the right credentials.

```ts title="Typed resolver"
const resolveAuth = createAuthResolver<RequestContext>((ctx) => {
  if (ctx.isBrowser && ctx.nonce) return { nonce: ctx.nonce, credentials: 'include' };
  if (ctx.token) return { token: ctx.token };
  return null;
});

const auth = await resolveAuth(context);
const post = await wp.request({ endpoint: '/posts/1', auth: auth ?? undefined });
```

Use resolvers when auth depends on who is making the request, where it runs, or how credentials are stored.

## Framework examples

### Express (no resolver needed)

```ts title="Request-scoped client"
app.use(async (req, _res, next) => {
  req.wp = new WordPressClient({
    baseUrl: process.env.WP_BASE_URL!,
    auth: req.user?.wpJwt ? { token: req.user.wpJwt } : undefined,
  });
  next();
});
```

### Generic SSR handler

```ts title="Auth from request cookies/headers"
import { WordPressClient, createAuthResolver, resolveWordPressAuth, type RequestAuthResolver } from 'fluent-wp-client';

const requestAuth: RequestAuthResolver = createAuthResolver((request) => {
  const jwt = request.headers.get('cookie')?.match(/wp_jwt=([^;]+)/)?.[1];
  if (jwt) return { token: decodeURIComponent(jwt) };
  if (process.env.WP_APP_PASSWORD) return { username: 'admin', password: process.env.WP_APP_PASSWORD };
  return null;
});

export async function handleRequest(request: Request) {
  const auth = await resolveWordPressAuth(requestAuth, request);
  const wp = new WordPressClient({ baseUrl: 'https://example.com', auth });
  return wp.users().me();
}
```

### Astro Actions

```ts title="Astro Actions"
import type { ActionAPIContext } from 'astro:actions';
import { createAuthResolver, resolveWordPressAuth } from 'fluent-wp-client';

const actionAuth = createAuthResolver<ActionAPIContext>((ctx) => {
  const jwt = ctx.cookies.get('wp_jwt')?.value;
  if (jwt) return { token: jwt };
  if (process.env.WP_APP_PASSWORD) return { username: 'admin', password: process.env.WP_APP_PASSWORD };
  return null;
});
```

### Next.js App Router

```ts title="Next.js route handler"
import { WordPressClient, createAuthResolver, resolveWordPressAuth } from 'fluent-wp-client';

const nextAuth = createAuthResolver<{ request: Request; params: { slug: string } }>(({ request }) => {
  const bearer = request.headers.get('authorization');
  if (bearer) return bearer;
  const jwt = request.headers.get('cookie')?.match(/wp_jwt=([^;]+)/)?.[1];
  return jwt ? { token: decodeURIComponent(jwt) } : null;
});

export async function GET(request: Request, { params }: { params: { slug: string } }) {
  const auth = await resolveWordPressAuth(nextAuth, { request, params });
  const wp = new WordPressClient({ baseUrl: process.env.WORDPRESS_BASE_URL!, auth });
  return Response.json(await wp.content('posts').item(params.slug));
}
```

### HMAC/signature auth

```ts title="Per-request signature"
const wp = new WordPressClient({
  baseUrl: 'https://example.com',
  authHeaders: ({ method, url, body }) => ({
    Authorization: createSignedAuthHeader({ method, url: url.toString(), body }),
    'X-Signature-Timestamp': new Date().toISOString(),
  }),
});
```

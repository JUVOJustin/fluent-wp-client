## Overview

Beyond built-in relations (`author`, `categories`, `tags`, `terms`, `featuredMedia`, `parent`), you can register custom relations for ACF fields, plugin embeds, custom meta references, and shared `_links`/`_embedded` data.

## How it works

Custom relations follow two-phase resolution:

1. **Extract from `_embedded`** — e.g., `_embedded['acf:post']`
2. **Fallback API calls** — if embedded data is missing, resolve from ID fields

```typescript
import { customRelationRegistry, type CustomRelationConfig } from 'fluent-wp-client';

const myRelation: CustomRelationConfig<MyType[]> = {
  name: 'myRelation',
  embeddedKey: 'my-plugin:items',
  extractEmbedded: (data) => Array.isArray(data) ? data.map(transformItem) : null,
  fallbackResolver: {
    resolve: async (client, post) => {
      const ids = post.meta?.my_relation_ids;
      return ids?.length ? fetchMyItems(ids) : [];
    },
  },
  requiredFields: ['meta'],
};

customRelationRegistry.register(myRelation);
```

## ACF field helpers

ACF `relationship`, `post_object`, and `taxonomy` fields have dedicated factories:

```typescript
import {
  createAcfRelationshipRelation,
  createAcfPostObjectRelation,
  createAcfTaxonomyRelation,
  customRelationRegistry,
} from 'fluent-wp-client';

customRelationRegistry.register(createAcfRelationshipRelation({
  relationName: 'linkedArticles',
  fieldName: 'acf_related_posts',
}));

customRelationRegistry.register(createAcfPostObjectRelation({
  relationName: 'primaryArticle',
  fieldName: 'acf_featured_post',
  multiple: false,
}));

customRelationRegistry.register(createAcfTaxonomyRelation({
  relationName: 'linkedGenres',
  fieldName: 'acf_related_genres',
  resource: 'genre',
  multiple: true,
}));

// Use alongside built-in relations
const post = await client
  .content('posts')
  .item('my-post')
  .with('author', 'categories', 'linkedArticles', 'primaryArticle', 'linkedGenres');
```

### ACF PHP configuration

:::caution[Required ACF setup]
ACF fields must have `show_in_rest => true` and `return_format => 'object'` for relation hydration to work.
:::

```php
[
  'name' => 'acf_related_posts',
  'type' => 'relationship',
  'return_format' => 'object',
  'show_in_rest' => true,
]
```

### Why field-based, not app-based

ACF groups all relational links under shared keys like `_links['acf:post']`. The helpers disambiguate by reading IDs from `acf[fieldName]` and filtering `_embedded` results to just those IDs. This prevents one `relationship` field from hydrating items belonging to another `post_object` field.

### ACF helper options

| Factory | Required params | Notes |
| --- | --- | --- |
| `createAcfRelationshipRelation` | `relationName`, `fieldName` | Optional `resource` for same-resource fallback |
| `createAcfPostObjectRelation` | `relationName`, `fieldName`, `multiple` | `post_object` can be single or multi-select |
| `createAcfTaxonomyRelation` | `relationName`, `fieldName`, `resource`, `multiple` | `resource` is the taxonomy REST base |

## Generic factories

For non-ACF plugins with ID-based relations:

```typescript
import { createIdCollectionRelation, createIdSingleRelation, resolvePostReferences } from 'fluent-wp-client';

const relatedPosts = createIdCollectionRelation({
  name: 'relatedPosts',
  embeddedKey: 'my-plugin:post',
  extractEmbeddedItem: (item) => {
    if (typeof item !== 'object' || !item) return null;
    return typeof (item as any).id === 'number' ? item : null;
  },
  getIds: (post) => (post as any).meta?.related_post_ids,
  resolveMany: resolvePostReferences,
  requiredFields: ['meta'],
});

customRelationRegistry.register(relatedPosts);
```

### Shared embed bucket factories

When a plugin exposes multiple fields through one shared `_embedded` key, use the linked-embedded factories:

```typescript
import { createLinkedEmbeddedCollectionRelation, resolveContentReferences } from 'fluent-wp-client';

const linkedArticles = createLinkedEmbeddedCollectionRelation({
  name: 'linkedArticles',
  embeddedKey: 'my-plugin:item',
  linksKey: 'my-plugin:item',
  extractItem: (item) => typeof (item as any)?.id === 'number' ? item : null,
  getIds: (post) => (post as any).meta?.linked_article_ids,
  resolveMany: (client, ids) => resolveContentReferences(client, 'posts', ids),
  requiredFields: ['meta', '_links'],
});
```

These factories read IDs first, filter `_embedded` to matching items, follow `_links` for missing items, then fall back to resource fetchers.

## Custom relation examples

### WooCommerce products

```typescript
import { customRelationRegistry, createArrayExtractor } from 'fluent-wp-client';

const wooProducts: CustomRelationConfig<WooProduct[]> = {
  name: 'relatedProducts',
  embeddedKey: 'woocommerce:related_products',
  extractEmbedded: createArrayExtractor((item) => {
    const obj = item as Record<string, unknown>;
    return typeof obj?.id === 'number'
      ? { id: obj.id, name: String(obj.name ?? ''), price: String(obj.price ?? '') }
      : null;
  }),
  fallbackResolver: {
    resolve: async (client, post) => {
      const ids = (post as any).meta?.related_product_ids ?? [];
      return Promise.all(ids.map(async (id: number) => {
        const { data } = await client.request({ endpoint: `/wp-json/wc/v3/products/${id}`, method: 'GET' });
        return data as WooProduct;
      })).then(results => results.filter(Boolean));
    },
  },
  requiredFields: ['meta'],
};

customRelationRegistry.register(wooProducts);
```

### Plugin embed (no fallback)

```typescript
customRelationRegistry.register({
  name: 'customPluginData',
  embeddedKey: 'my-plugin:custom-data',
  extractEmbedded: createArrayExtractor((item) => {
    const obj = item as Record<string, unknown>;
    return typeof obj?.id === 'number' ? { id: obj.id, value: String(obj.value ?? '') } : null;
  }),
});
```

## Extractor helpers

- `createArrayExtractor<T>(transform)` — array of items, null items filtered out
- `createSingleExtractor<T>(transform)` — single item or null
- `extractEmbeddedData<T>(post, key)` — low-level `_embedded` access

## Registry API

```typescript
customRelationRegistry.register(relation);
customRelationRegistry.has('name');
customRelationRegistry.get('name');
customRelationRegistry.unregister('name');
customRelationRegistry.getAllNames();
customRelationRegistry.clear();
```

## Type safety

:::note
Built-in relations are fully typed. Custom relations are `unknown` in `post.related` — cast at the call site:
```typescript
const products = post.related.relatedProducts as WooProduct[];
```
:::

## Best practices

:::tip
1. Always provide fallback resolvers — `_embedded` may not be available.
2. Return `null` or `[]` for missing data, never throw.
3. Specify `requiredFields` to preserve critical data.
4. Clean up with `customRelationRegistry.clear()` in tests.
:::

## Migration from manual extraction

```typescript
// Before: manual ACF hydration
const ids = post.acf?.acf_related_posts;
const related = await Promise.all(ids?.map(id => client.content('posts').item(id)) ?? []);

// After: register once, use everywhere
customRelationRegistry.register(createAcfRelationshipRelation({
  relationName: 'linkedArticles', fieldName: 'acf_related_posts',
}));
const post = await client.content('posts').item(123).with('linkedArticles');
```

## Performance

:::note
- Embedded data is fastest (no extra API calls)
- Fallback resolvers add latency — consider caching
- Multiple custom relations may trigger separate API calls
:::

`fluent-wp-client` is a runtime-agnostic TypeScript client for the WordPress REST API. It works on **Node.js**, **Deno**, **Bun**, and in the **browser** using only web-standard APIs (`fetch`, `URL`, `Blob`).

## Install

```bash
npm install fluent-wp-client
```

## Quick start

<Steps>
1. Install
  ```bash
  npm install fluent-wp-client
  ```

2. Create a client
  ```ts
  import { WordPressClient } from 'fluent-wp-client';

  const wp = new WordPressClient({
    baseUrl: 'https://your-wordpress-site.com',
  });
  ```

3. Interact with the API
   ```ts 
  const posts = wp.content('posts');

  // Read a list of posts
  const recentPosts = await posts.list({ perPage: 10 });

  // Create a draft post (requires auth)
  const draft = await posts.create({ title: 'Hello', status: 'draft' });

  // Custom post type example
  const books = wp.content('books');
  const allBooks = await books.listAll();

  // Get item directly
  const bySlug = await books.item('my-book');

   // Get the item with relations
   const hydrated = await books.item('my-book').with('author', 'terms');

   // Execute a registered WordPress ability
   const result = await wp.executeRunAbility('my-plugin/send-notification', { userId: 123 });
    ```
</Steps>

:::tip[What is a Resource Client?]

When you call `wp.content('posts')` or `wp.terms('categories')`, you get a **resource client**. Think of it as a dedicated helper for working with one type of content in WordPress.

- `wp.content('posts')` returns a client for posts
- `wp.content('books')` returns a client for your custom book post type
- `wp.terms('categories')` returns a client for categories

Use the client to list items, create new ones, update, or delete. The actual data you get back—a post, book, or category—is a **DTO (Data Transport Object)** which is fully serializable.

:::

## Features

- **Unified typed content builders** — `content('posts')`, `content('pages')`, `content('books')`, and `terms('genre')` share one API shape, with stricter typing for built-in resources
- **Cross-resource search** — `searchContent()` queries across posts, pages, and CPTs via the `/wp/v2/search` endpoint
- **Flexible CPT defaults** — generic content reads tolerate post types that omit `title`, `content`, `excerpt`, or `author`
- **Gutenberg block parsing** — single-item queries expose `.getBlocks()` and `.getContent()`
- **Auth flexibility** — Basic auth (application passwords), JWT, cookie+nonce, prebuilt headers, and per-request signing
- **WordPress Abilities API** — discover and execute registered abilities with optional schema validation
- **Schema discovery** — fetch live JSON Schemas for any resource or ability at runtime, convert to Zod with `z.fromJSONSchema()`, and use directly for input/output validation
- **Standard Schema validation** — validator-agnostic root exports; native Zod available from `fluent-wp-client/zod`
- **Extensible relation API** — fluent relations for posts and custom entities, plus generic ID-backed and shared link/embed relation factories

## Development notes

Contributor-facing transport, mutation helpers, query primitives, and base resource classes live in `src/core/`, while relation contracts, definition factories, and the fluent relation builder live in `src/builders/`. Package consumers continue to import the public API from `src/index.ts`.

## Overview

`fluent-wp-client` derives its discovery catalog from the REST schema returned by WordPress `OPTIONS` requests. When ACF schema metadata is available in that catalog, read it with `getSchemaValue()` instead of hard-coding individual field paths in UI components.

ACF does not expose choice labels in its REST schema by default. Add the keys you want to discover with `acf/rest/get_field_schema`:

```php title="mu-plugins/fluent-wp-client-acf-support.php"
<?php
/**
 * Adds ACF field descriptions and choice labels to the REST schema.
 */
add_filter(
    'acf/rest/get_field_schema',
    static function ( array $schema, array $field ): array {
        if ( ! empty( $field['description'] ) ) {
            $schema['description'] = $field['description'];
        }

        $choice_field_types = [ 'select', 'radio', 'checkbox', 'button_group' ];

        if (
            isset( $field['type'], $field['choices'] )
            && in_array( $field['type'], $choice_field_types, true )
            && is_array( $field['choices'] )
        ) {
            $schema['choices'] = array_map(
                static fn ( $value, $label ): array => [
                    'value' => (string) $value,
                    'label' => (string) $label,
                ],
                array_keys( $field['choices'] ),
                array_values( $field['choices'] )
            );
        }

        return $schema;
    },
    10,
    2
);
```

For UI code that does not know each field type ahead of time, fetch the full ACF schema field map once and build a reusable choice-label lookup.

```ts title="Building a choice-label lookup for all ACF fields"
type AcfChoice = {
  label: string;
  value: string;
};

type AcfSchemaField = {
  choices?: AcfChoice[];
};

const acfFields = await wp.content("posts").getSchemaValue<
  Record<string, AcfSchemaField>
>(
  "properties.acf.properties",
);

const choiceLabels = new Map<string, Map<string, string>>();

for (const [fieldName, fieldSchema] of Object.entries(acfFields ?? {})) {
  if (!Array.isArray(fieldSchema.choices)) continue;

  choiceLabels.set(
    fieldName,
    new Map(
      fieldSchema.choices.map((choice) => [choice.value, choice.label]),
    ),
  );
}
```

Use that lookup while rendering ACF values. Fields without `choices` automatically fall back to their raw value.

```ts title="Rendering ACF values with labels when available"
function getAcfDisplayValue(fieldName: string, value: unknown): string {
  if (typeof value === "string") {
    return choiceLabels.get(fieldName)?.get(value) ?? value;
  }

  return String(value);
}

const post = await wp.content("posts").item("test-post-001");
const acf = post.acf && typeof post.acf === "object" ? post.acf : {};

const renderedFields = Object.entries(acf).map(([fieldName, value]) => ({
  fieldName,
  value,
  label: getAcfDisplayValue(fieldName, value),
}));
```

AI SDK integrations can still use `describeResourceTool` to expose the same descriptions and choice labels to a model, while read tools use the discovered field paths for validation.

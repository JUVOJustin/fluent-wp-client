## Overview

Schema discovery lets you fetch the JSON Schema WordPress exposes for any resource or ability, then use it directly for validation — without hard-coding schemas upfront.

If you want generated artifacts instead of runtime discovery, use the built-in CLI:

```bash title="Generating schema artifacts"
npx fluent-wp-client schemas --url https://example.com
npx fluent-wp-client schemas --format both --json-out wp-schemas.json --zod-out wp-schemas.ts
npx fluent-wp-client types --url https://example.com
```

:::tip[When to use schema discovery]
This is especially useful for generic integrations, AI agents, and tools that work across multiple sites where the schema shape is not known at build time.
:::

## Describe a resource

Call `.describe()` on any resource client to get its JSON Schemas:

```ts
const desc = await wp.content('books').describe();

desc.schemas.item       // single item response shape
desc.schemas.collection // collection response shape
desc.schemas.create     // accepted fields for create
desc.schemas.update     // accepted fields for update (no required fields)
```

Works the same for terms, first-class resources, and abilities:

```ts
await wp.terms('categories').describe();
await wp.media().describe();
await wp.users().describe();
await wp.comments().describe();
await wp.settings().describe();
await wp.ability('my-plugin/send-notification').describe();
```

## Validate with discovered schemas

Convert a discovered schema to Zod with `z.fromJSONSchema()` and pass it to `create()`, `update()`, or `.inputSchema()`:

```ts title="Content"
import { z } from 'zod';

const desc = await wp.content('books').describe();
const createSchema = z.fromJSONSchema(desc.schemas.create!);

// Valid input succeeds
const book = await wp.content('books').create(
  { title: 'My Book', status: 'publish' },
  createSchema,
);

// Invalid input throws WordPressSchemaValidationError before the request
await wp.content('books').create({ content: 'no title' }, createSchema);
```

```ts title="Ability"
const desc = await wp.ability('my-plugin/send-notification').describe();
const inputSchema = z.fromJSONSchema(desc.schemas.input!);

const result = await wp
  .ability('my-plugin/send-notification')
  .inputSchema(inputSchema)
  .run({ userId: 42 });
```

## Explore: full catalog

`wp.explore()` returns schemas for all content types, taxonomies, first-class resources, and abilities in one call. The result is a plain serializable object.

```ts
const catalog = await wp.explore();

// catalog.content   — post types (posts, pages, books, …)
// catalog.terms     — taxonomies (categories, tags, …)
// catalog.resources — media, users, comments, settings
// catalog.abilities — registered WordPress abilities

const schema = z.fromJSONSchema(catalog.content.books.schemas!.create!);
```

Limit the scope with `include`, or bust the in-memory cache with `refresh`:

```ts
await wp.explore({ include: ['content', 'terms'] });
await wp.explore({ refresh: true });
```

Available kinds: `'content'`, `'terms'`, `'resources'`, `'abilities'`.

## Authentication

:::caution
Use an authenticated client to get full schema coverage. Write schemas (`create`, `update`) are only available to users with appropriate capabilities.
:::

```ts
const wp = new WordPressClient({
  baseUrl: 'https://example.com',
  auth: { username: 'admin', password: process.env.WP_APP_PASSWORD },
});

const usersSchema = await wp.users().describe();
const settingsSchema = await wp.settings().describe();
```

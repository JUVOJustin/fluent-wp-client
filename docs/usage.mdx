## Method reference

### Content and term builders

```
content('posts') / content('pages') / content(resource, schema?)
terms('categories') / terms('tags') / terms(resource, schema?)
```

Each builder exposes:
- `.list(filter?)` — one page
- `.listAll(filter?)` — auto-paginates all pages
- `.listPaginated(filter?)` — returns `{ data, total, totalPages, page, perPage }`
- `.item(idOrSlug)` — single item (awaitable)
- `.create()` / `.update()` / `.delete()` — mutations
- `.describe()` — JSON Schema discovery

`content(resource).item(idOrSlug)` also supports `.getContent()` for `{ raw, rendered, protected }` and `.with(...)` for relation hydration.

:::caution
`.getContent()` uses `context=edit` and requires edit capabilities.
:::

Passing a schema to `content(resource, schema)` or `terms(resource, schema)` validates all reads and mutations on that resource client.

### First-class resources

| Resource | Extras |
| --- | --- |
| `media()` | `.upload()`, `.getImageUrl()`, `.with(...)` on items |
| `users()` | `.me()`, `.with(...)` on items |
| `comments()` | `.with(...)` on items |
| `settings()` | `.get()`, `.update()`, `.describe()` (singleton) |

All collection resources share `.list()`, `.listAll()`, `.listPaginated()`, `.item()`, `.create()`, `.update()`, `.delete()`, `.describe()`.

### Search

```ts
const results = await wp.searchContent('hello');
const pages = await wp.searchContent('about', { type: 'post', subtype: 'page' });
const mixed = await wp.searchContent('test', { subtype: ['post', 'page', 'book'] });
```

Returns `{ id, title, url, type, subtype }[]`.

### Extensible collection filters

Typed core filters accept extra unknown keys. CamelCase names become snake_case query params automatically:

```ts
const posts = await wp.content('posts').list({
  search: 'hello',
  include: [4, 8],
  titleSearch: 'Hello',  // becomes title_search
});
```

### Request-scoped headers

All read and mutation helpers accept optional custom headers:

```ts
const posts = await wp.content('posts').list(
  { perPage: 5 },
  { headers: { 'x-custom-header': 'value' } },
);
```

:::caution[Auth overrides are blocked]
To use different auth credentials, create a separate `WordPressClient` instance.
:::

### Auth and ability helpers

- `loginWithJwt(credentials)` / `validateJwtToken(token?)`
- `getAbilities()` / `getAbility(name)` / `getAbilityCategories()`
- `executeGetAbility()` / `executeRunAbility()` / `executeDeleteAbility()`
- `ability<TInput, TOutput>(name)` — fluent builder

### Transport

- `request(options)` / `fetchAPI(endpoint, params?)` / `fetchAPIPaginated(endpoint, params?)`
- `setHeaders(name, value)` / `hasAuth()`
- `route(resource, namespace?)` / `namespace('my-plugin/v1').route('sync')`

## Relation hydration

### `embed: true` vs `.with()`

```ts
// Raw WordPress _embedded — untyped, manual parsing
const posts = await wp.content('posts').list({ perPage: 10, embed: true });
posts[0]._embedded['wp:term'] // mixed categories + tags

// Typed relation hydration — clean API with fallbacks
const post = await wp.content('posts').item('hello-world').with('author', 'categories', 'tags');
post.related.author     // WordPressAuthor | null
post.related.categories // WordPressCategory[]
```

:::tip[Prefer `.with()`]
Type safety, clean names, automatic taxonomy separation, and fallback API calls when `_embedded` is missing.
:::

| Feature | `embed: true` | `.with()` |
|---------|--------------|-----------|
| Type safety | Manual casting | Fully typed |
| Names | `wp:term`, `wp:featuredmedia` | `categories`, `featuredMedia` |
| Fallback | None | Automatic API calls from ID fields |
| Custom relations | Manual | Register once, use everywhere |

You can combine both: `.item('slug', { embed: true }).with('author')` gives you raw `_embedded` AND typed `related`.

### Internals and edge cases

`.with()` internally requests `_embed`, hydrates `related`, and strips `_embedded` from the output. If `_embedded` is missing, it falls back to API calls using ID fields.

```ts
// Missing author ID → null (not an error)
const post = await publicClient.content('posts').item('slug').with('author');

// No categories → empty array
post.related.categories // []
```

### Built-in relation names

`author`, `categories`, `tags`, `terms`, `featuredMedia`, `parent`

Custom taxonomies are at `terms.taxonomies.<taxonomy>`. For ACF and plugin relations, see the [extensible relations guide](./extensible-relations.mdx).

### Post-like typing

```ts
import type { WordPressCustomPost, WordPressPostBase } from 'fluent-wp-client';

type Artifact = WordPressCustomPost<{ type: 'artifact' }>;     // sparse — optional fields
type ContentBook = WordPressPostBase & { type: 'book' };       // strict — requires content fields
```

## CRUD walkthrough

### Posts and pages

```ts
const posts = wp.content('posts');
const created = await posts.create({ title: 'My post', status: 'draft' });
await posts.update(created.id, { title: 'Updated', status: 'publish' });
await posts.delete(created.id, { force: true });
```

### Terms

```ts
const cat = await wp.terms('categories').create({ name: 'News' });
await wp.terms('categories').update(cat.id, { name: 'Company News' });
await wp.terms('categories').delete(cat.id, { force: true });
```

### Comments

```ts
const comment = await wp.comments().create({ post: 42, content: 'Nice!', status: 'approve' });
await wp.comments().update(comment.id, { content: 'Updated.' });
await wp.comments().delete(comment.id, { force: true });
```

### Media

```ts
const media = await wp.media().upload({
  file: imageBlob, filename: 'cover.jpg', mimeType: 'image/jpeg',
  title: 'Cover', alt_text: 'Book cover',
});
await wp.media().delete(media.id, { force: true });
```

### Users and settings

```ts
const me = await wp.users().me();
const settings = await wp.settings().get();
await wp.settings().update({ title: 'New Title' });
```

For CPT and custom taxonomy CRUD, see the [custom endpoints guide](./custom-endpoints.mdx).

## Validation

Mutations auto-validate with built-in schemas. Pass your own Standard Schema-compatible validator to override:

```ts
import * as v from 'valibot';

const created = await wp.content('posts').create(
  { title: 'Validated', status: 'draft' },
  v.object({ id: v.number(), slug: v.string(), status: v.string() }),
);
```

See the [validation guide](./validation.mdx) for Zod extension, Valibot, sparse CPT, and Astro patterns.

## Gutenberg blocks

```ts
import { withBlocks } from 'fluent-wp-client/blocks';

const wpBlocks = withBlocks(wp);
const blocks = await wpBlocks.content('posts').item('hello-world').blocks().get();
```

See the [Gutenberg content guide](./gutenberg-content.mdx) for the full workflow.

## Pagination

:::note[WordPress caps `per_page` at 100]
Use `.listAll()` for complete datasets. Use `.listPaginated()` for totals and page metadata.
:::

## Error handling

- `WordPressApiError` — REST API errors with status, code, and payload
- `WordPressSchemaValidationError` — validation failures
- Cross-origin absolute URLs are rejected before requests are sent
